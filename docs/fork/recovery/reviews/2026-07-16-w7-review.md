# W7 review

## Bottom line

W7 should **not** be implemented as one omnibus cleanup. The original plan grouped three unrelated optional refactors, and the final architecture review added five concrete LOW findings. Split it by authority and behavioral risk.

## Scope drift

**Original W7** (`RECOVERY_PLAN.md:130-132`):
1. R12 shared terminal-emission helper.
2. R03A centralized advertising-subscribe/verdict consumption.
3. R02 `sidecar.rs` / `provider/mod.rs` size-debt reduction.

**Final architecture findings** (`RECOVERY_PLAN.md:413-433`):
1. Duplicated provider-evidence sequencing across both turn engines.
2. Private `TurnInterruptedError` forces string-based cross-module identification.
3. Detached cancellation can end as `failed` or `stopped` depending on update order.
4. `ClassifiedEvidenceError` does not expose its inner source.
5. `append_progress_provenance` grows without a bound.

## 30/30 evidence matrix

### 1. Provider-evidence consolidation
- **Source:** `agent/turn_loops.rs` has 6 error-emission calls; `agent/turn_streaming_mpsc.rs` has 8; shared event construction is in `agent/evidence.rs:143-170`.
- **Current behavior:** both engines emit correlated request/response/turn records. Eleven `r12_` fixtures pin success, transport error, stream error, cancellation, context-limit retry, and secret-safe classification.
- **Missing fixture:** an explicit parity assertion that a helper extraction leaves the full event matrix and correlation IDs unchanged.
- **Impact:** maintainability and R09 file-size debt. No current correctness defect.
- **Regression surface:** medium-high. Blocking and MPSC engines intentionally differ in cancellation and streaming control flow.
- **Smallest safe patch:** extract only event construction/common classification. Do not unify the two async control loops or hide engine-specific cancellation branches.

### 2. Typed interruption predicate
- **Source:** private `TurnInterruptedError` at `agent/evidence.rs:226-235`; server fixture compares the display string at `client_lifecycle_tests.rs:479-483`.
- **Current behavior:** agent-local classification is typed; server-side consumers cannot distinguish interruption without the string `turn interrupted`.
- **Missing fixture:** predicate recognizes the real marker through an `anyhow` chain and rejects a lookalike string error.
- **Impact:** maintainability today, prerequisite for deterministic status normalization.
- **Regression surface:** low if the marker stays private.
- **Smallest safe patch:** expose `Agent::error_is_turn_interruption(&anyhow::Error) -> bool` as crate-visible. Do not expose the marker type.

### 3. Cancelled-turn status normalization
- **Source:** `client_lifecycle.rs:784-798` and `client_actions.rs:1124-1138` map every `Err` to `failed`; `cancel_processing_message` writes `stopped/cancelled` at `client_lifecycle.rs:3043-3058`.
- **Current behavior:** in the `NO_LOCAL_TASK` path, whichever update lands last determines `failed` versus `stopped`. Both are terminal for reclaim, so liveness is correct.
- **Missing fixture:** deterministic detached-cancel coverage proving the final member label is `stopped` regardless of completion/cancel ordering.
- **Impact:** operator/UI consistency only unless a future consumer distinguishes the labels.
- **Regression surface:** medium-low because two completion consumers must agree.
- **Smallest safe patch:** use the typed predicate in both consumers. Map interrupted errors to `stopped` with `cancelled` detail; retain `failed` for all other errors.

### 4. Error source-chain preservation
- **Source:** empty `Error` implementation for `ClassifiedEvidenceError` at `agent/evidence.rs:260-272`; retry metadata consumers downcast at `client_lifecycle.rs:798` and `client_actions.rs:1136`.
- **Current behavior:** wrapped errors lose source traversal. The current retry-after-bearing stream path remains unwrapped, so no observed breakage exists.
- **Missing fixture:** wrap a `StreamError` with retry metadata, verify `source()`/downcast traversal and `retry_after_secs` preservation.
- **Impact:** latent correctness and diagnostics hardening.
- **Regression surface:** very low.
- **Smallest safe patch:** implement `Error::source()` by returning the inner `anyhow::Error` as an error trait object.

### 5. Bounded progress provenance
- **Source:** `jcode-plan/src/lib.rs:70-79`; unbounded direct takeovers at `comm_control.rs:1741-1750`; dead-member reclaims are capped but also append provenance.
- **Current behavior:** the original checkpoint is preserved and every takeover/reclaim note is appended. Direct takeovers have no count or byte cap.
- **Missing fixture:** repeated takeovers exceed the chosen bound while retaining the original checkpoint and newest provenance.
- **Impact:** bounded persistence/broadcast size and protection against churn-driven growth.
- **Regression surface:** medium because truncation policy can erase useful recovery history.
- **Smallest safe patch:** choose an explicit 2 KiB UTF-8-safe policy that preserves the original checkpoint plus the newest notes, with a visible omission marker. Do not simply keep the tail.

## Recommended sequence

1. **W7a, R12 error semantics:** add `ClassifiedEvidenceError::source` and the typed interruption predicate with focused tests.
2. **W7b, R04/R05B status behavior:** normalize interrupted turns to `stopped/cancelled` in both server consumers; add a deterministic status fixture.
3. **W7c, R12 pure refactor:** narrow provider-evidence helper extraction, guarded by all 11 R12 fixtures and exact event-count/correlation parity.
4. **W7d, R05A/R05B persistence policy:** bound progress provenance only after agreeing on the retention rule.
5. **Keep dormant:** R03A verdict centralization and R02 file-size splitting. Neither has a new correctness trigger, and R09 forbids blanket cleanup.

## Recommendation

Proceed with W7a first. It is the smallest, safest slice and creates the typed mechanism needed by W7b. Treat W7c as a genuine zero-behavior refactor. Treat W7d as a policy decision, not a casual truncation patch.
