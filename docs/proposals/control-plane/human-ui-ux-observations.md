# Human UI/UX observations

Owner: John

Use this log for human-facing observations about how a control plane should feel and behave.

## How to add an observation

```md
## YYYY-MM-DD - Short title

- Context:
- Observation:
- Friction or opportunity:
- Desired control affordance:
- Safety concern:
- Follow-up question:
```

## Seed prompts

- When do you want to see all active jcode work at once?
- Which controls need to be one-click, and which should require confirmation?
- What should be visible on mobile versus desktop versus TUI?
- What makes an agent feel stuck, risky, or trustworthy?
- What context is needed before approving a destructive or externally visible action?
